# CI セットアップ

このプロジェクトには以下の機能を持つ包括的なGitHub Actions CIパイプラインが含まれています：

## ワークフロージョブ

### 1. ESLint

- VueとTypeScriptサポートでESLintを実行
- Nuxt固有のルールに`@nuxt/eslint-config`を使用
- コマンド: `yarn lint`

### 2. TypeScript型チェック

- vue-tscで静的型解析を実行
- VueコンポーネントとTypeScriptファイル全体で型安全性を確保
- コマンド: `yarn type-check`

### 3. ユニットテスト (Vitest)

- Vitestによる高速ユニットテスト
- jsdom環境でVueコンポーネント用に設定
- コマンド: `yarn test:run`

### 4. セキュリティ監査

- 依存関係の既知の脆弱性をチェック
- `yarn audit`を中程度のレベル閾値で使用
- コマンド: `yarn audit`

### 5. ビルド検証

- アプリケーションが正常にビルドされることを確認
- ビルド成果物（`.output/`）をアーティファクトとしてアップロード
- 他のジョブで再利用するため、retention-daysを1日に設定
- コマンド: `yarn build`

### 6. E2Eテスト (Playwright)

- ビルドジョブからアーティファクトをダウンロード（再ビルド不要）
- アプリケーションがエラーなしで読み込まれることを検証するスモークテスト
- 主要なユーザーフローとページ機能をテスト
- コマンド: `yarn test:e2e`

### 7. CodeQLセキュリティ解析

- ビルドジョブからアーティファクトをダウンロード（再ビルド不要）
- GitHubの高度なセキュリティ解析
- セキュリティ脆弱性とコード品質の問題をスキャン
- pushとPRイベントで自動実行

### 8. アーティファクトクリーンアップ

- E2EテストとCodeQL解析の完了後に実行
- ビルドアーティファクトを自動削除してストレージを節約
- `if: always()`により、ジョブが失敗しても実行される

## トリガー

CIワークフローは以下の場合に実行されます：

- `master`と`develop`ブランチへのpush
- `master`と`develop`ブランチをターゲットとするプルリクエスト

## アーティファクト共有の最適化

このCIワークフローは、ビルドアーティファクトを複数のジョブ間で共有することで最適化されています：

### メリット

1. **ビルド時間の短縮**: ビルドは一度だけ実行され、E2EテストとCodeQL解析で再利用されます
2. **リソースの効率化**: 同じビルド処理を3回実行する代わりに1回のみ実行
3. **一貫性の向上**: すべてのジョブで同じビルド成果物を使用することで、環境差異を排除
4. **ストレージの節約**: アーティファクトは使用後すぐに自動削除され、1日間のみ保持

### 実装の詳細

- **buildジョブ**: `.output/`ディレクトリをアーティファクトとしてアップロード
- **e2e-tests & codeqlジョブ**: buildジョブの完了を待機（`needs: build`）してアーティファクトをダウンロード
- **cleanupジョブ**: すべてのジョブ完了後にアーティファクトを自動削除

## ローカル開発

すべてのCIチェックはローカルで実行できます：

```bash
# 依存関係のインストール
yarn install

# リンティングの実行
yarn lint

# 型チェックの実行
yarn type-check

# ユニットテストの実行
yarn test

# E2Eテストの実行（開発サーバーが必要）
yarn dev # 一つのターミナルで
yarn test:e2e # 別のターミナルで

# アプリケーションのビルド
yarn build

# 脆弱性のチェック
yarn audit
```

## 利用可能な追加ツール

- **Prettier**: コード整形 (`yarn format`)
- **Test UI**: インタラクティブテストランナー (`yarn test:ui`)
- **Coverage**: テストカバレッジレポート (`yarn test:coverage`)
